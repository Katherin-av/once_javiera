# Requerimientos de Software

## ¿Qué es un requerimiento?

Un requerimiento es una propiedad documentada y verificable que un sistema debe cumplir para resolver un problema o alcanzar un objetivo. Debe poder escribirse y comprobarse.

## Niveles de abstracción

Los requerimientos existen en diferentes niveles:

1. **Necesidad del usuario:** representa el problema o necesidad del usuario.  
2. **Requerimiento del sistema:** define lo que debe hacer el sistema.  
3. **Especificación técnica:** explica los detalles técnicos para su implementación.

## Fuentes de requerimientos

Los requerimientos pueden provenir de diferentes actores:

- Usuarios finales
- Clientes o patrocinadores
- Leyes y normas
- Sistemas externos

## Características de un buen requerimiento

Un requerimiento debe ser:

- Necesario
- No ambiguo
- Verificable
- Consistente
- Completo
- Atómico
- Trazable

## Requerimientos funcionales

Los requerimientos funcionales describen **qué hace el sistema**.

Ejemplos:

- Iniciar sesión
- Calcular promedios
- Guardar información
- Enviar notificaciones
- Generar reportes
- Validar datos

## Requerimientos no funcionales

Los requerimientos no funcionales describen **cómo funciona el sistema** y la calidad con la que opera.

Incluyen aspectos como:

- Rendimiento
- Seguridad
- Usabilidad
- Confiabilidad
- Escalabilidad
- Compatibilidad

Estos deben poder medirse mediante métricas claras.

## Atributos de calidad

La norma ISO 25010 define atributos importantes del software:

- Adecuación funcional
- Eficiencia de desempeño
- Compatibilidad
- Usabilidad
- Confiabilidad
- Seguridad
- Mantenibilidad
- Portabilidad

## Historias de usuario

Las historias de usuario describen necesidades desde la perspectiva del usuario.

Formato:

**Como** [usuario],  
**quiero** [acción],  
**para** [beneficio].

También deben incluir criterios de aceptación para comprobar si el requerimiento se cumple.

## Conclusión

Los requerimientos permiten definir correctamente qué debe hacer un sistema y cómo debe funcionar, ayudando a evitar errores y mejorando la calidad del software.
