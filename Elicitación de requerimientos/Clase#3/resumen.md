# Elicitación y Viabilidad de Requerimientos

## ¿Qué es la elicitación?

La elicitación de requerimientos es el proceso de descubrir las necesidades de clientes y usuarios para convertirlas en requerimientos formales. No consiste solo en recolectar información, sino en ayudar al cliente a identificar lo que realmente necesita.

El analista actúa como un detective: observa, hace preguntas y conecta información para entender el problema real.

## Técnicas de elicitación

Las principales técnicas son:

- **Entrevistas:** conversaciones para obtener información detallada.
- **Encuestas:** permiten recolectar datos de muchas personas rápidamente.
- **Observación:** consiste en ver cómo trabajan los usuarios para detectar necesidades que no expresan.

Estas técnicas se complementan entre sí.

## Habilidades del analista

Un buen analista necesita:

- Escucha activa
- Curiosidad
- Empatía
- Tomar notas
- Evitar asumir cosas sin confirmar

## Entrevistas

Las entrevistas son conversaciones guiadas para obtener información útil sobre el sistema.

### Tipos de entrevistas

- **Estructurada:** preguntas fijas y ordenadas.
- **No estructurada:** conversación libre.
- **Semi-estructurada:** mezcla preguntas guía con flexibilidad.

### Etapas

1. **Antes:** investigar y preparar preguntas.  
2. **Durante:** conversar, profundizar y tomar notas.  
3. **Después:** organizar la información obtenida.

### Tipos de preguntas

- **Abiertas:** permiten respuestas amplias.
- **Cerradas:** respuestas cortas o específicas.
- **De sondeo:** profundizan en un tema mencionado.

## Encuestas

Las encuestas sirven para recopilar información de muchas personas al mismo tiempo.

Se usan cuando:

- Hay muchos participantes.
- Se necesitan datos cuantitativos.
- Se buscan respuestas anónimas.

### Tipos de preguntas

- Selección única
- Selección múltiple
- Escala Likert
- Preguntas numéricas
- Preguntas abiertas

### Reglas para una buena encuesta

- Debe ser corta.
- Tener preguntas claras.
- Una sola idea por pregunta.
- Evitar lenguaje técnico.
- Probarla antes de aplicarla.

## Análisis de viabilidad 

La viabilidad permite evaluar si un requerimiento realmente puede realizarse.

### Dimensiones de viabilidad

- **Técnica:** disponibilidad de tecnología y conocimientos.
- **Económica:** presupuesto suficiente.
- **Operativa:** aceptación y utilidad para usuarios.
- **Legal:** cumplimiento de normas y leyes.
- **Temporal:** posibilidad de terminar en el tiempo disponible.

## Conclusión

La elicitación ayuda a descubrir las necesidades reales del cliente, mientras que el análisis de viabilidad permite evaluar si esas necesidades pueden desarrollarse de manera realista.
